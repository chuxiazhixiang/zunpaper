➕ 加一个论文分类

想追踪一个新方向（比如「医疗机器人」）？填下面几项，点 「生成并下载配置文件」，再把下载到的文件上传到你的仓库 config/channels.d/ 文件夹，重新构建就会多出这个分类，并把最近一个月里符合条件的论文一次性收进来。 全程不用写代码、不用命令行。

示例高质量论文（可选，最多 6 篇） 贴几篇你认可的代表作 arXiv 链接 —— 它们会立即上墙，并作为 AI 判定的「正例参考」，让收录更准。

📄 配置文件预览

下载的文件名将是 your-id.yaml

填好左侧表单，这里会实时生成配置内容…

🚀 怎么用（3 步，不用命令行）

  1. 下载上面生成的 <id>.yaml 文件。
  2. 打开你的 GitHub 仓库 → 进入 config/channels.d/ 文件夹 → Add file ▸ Upload files → 把文件拖进去 → Commit changes。 （没有 channels.d 文件夹？在 config/ 里 Add file ▸ Create new file，文件名输入 channels.d/<id>.yaml，把内容粘进去再 Commit。）
  3. 触发构建：仓库 Actions 页 ▸ 选 daily ▸ Run workflow（或等下一次自动构建）。 构建完成后刷新首页，顶部就会出现你的新分类 tab，并把最近一个月符合条件的论文收了进来。

想调整？随时改这个文件里的 judge_prompt / 关键词再 Commit，下次构建会自动对相关论文重新判定。 删掉文件则该分类下次构建下架。
