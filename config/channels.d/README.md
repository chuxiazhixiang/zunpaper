# 自定义分类（拖入式）

把一个 `.yaml` 文件丢进这个文件夹，**一个文件 = 一个新分类**，重新 build 后主页顶部就会多出这个分类的 tab。不用改任何现有文件，零格式冲突。

最简单的做法：打开网站的 **「➕ 加分类」** 页面，按提示填几项，点「下载配置文件」，把下载到的 `<id>.yaml` 上传到这个文件夹即可（GitHub 网页上传也行，不用装命令行）。

## 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `id` | ✓ | 分类英文小写 slug（URL 用），如 `medical-robotics` |
| `name` | ✓ | 分类显示名，如 `医疗机器人` |
| `emoji` | | tab 上的小图标，如 `"🩺"` |
| `arxiv_categories` | | 要扫的 arXiv 分类码，默认 `[cs.RO]` |
| `keywords` | ✓ | 标题/摘要命中任一即「候选」（OR 关系，决定召回 + 预筛） |
| `exclude` | | 命中任一即排除 |
| `desc` | 推荐 | 一句话方向定义（人 + AI 都看） |
| `judge_prompt` | 推荐 | 你自己写的「什么算 / 不算这个方向」。**这是本分类专属的独立 AI 判定标准，不受核心 6 类黑名单（如「不要医疗」）影响。** |
| `venues` | | 你关注的会议 / 期刊，命中给评分加分 |
| `examples` | | 几篇你认可的高质量论文（`{title, url}`）→ 立即上墙 + 当 AI 判定的正例参考 |
| `backfill_days` | | 首次激活时回填最近 N 天的 arXiv（默认 30；设 0 不额外回填） |
| `max_per_day` | | 每次每频道软上限 |

## 工作原理（B 方案：每类独立判定）

核心 6 类共用一段大 prompt（带「不要医疗」等全局黑名单）。你新加的分类**不走那段**，而是用你写的 `judge_prompt` **独立判一次**——只对命中本分类 `keywords` 的论文判定，互不干扰，所以能天然收录医疗这种被核心黑名单排除的方向。

> 改了 `judge_prompt` / `desc`？下次 build 会自动对相关论文重判（缓存按 prompt 指纹失效）。删掉这个文件？该分类及其专属论文会在下次 build 下架。
